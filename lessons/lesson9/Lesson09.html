Introduction to Lesson 9

Visualizations created using ggplot2 shouldn’t be simple (otherwise, just use base R plot() or qplot() – quick plot from ggplot2). They should have a richness that is not easily programmed in plot() – mostly because that richness can be achieved with the addition of a few simple options.

Lesson 9 is about taking the basic statistical graphics that you’ve seen in your first stats course, re-doing them using ggplot, and then using ggplot’s many additional features to polish them for presentation. One of the awards given at DataFest is “Best Data Visualization” and ggplot has been used by most of the winners.

Next, we create new variables from existing variables. This is done because the data may not be in the correct format for plotting, so we use our data management skills to make necessary modifications:

test <- test %>% 
  mutate(weight = as.numeric(Wt),
         height = (as.numeric(substr(test$Ht, start = 1, stop = 1)) * 12) +
           (as.numeric(substr(test$Ht, start = 3, stop = 4))))

We are basically modifying the data frame test by adding two new variables, weight and height. Both weight and height were saved as character variables, all we are doing is transforming them.

The function substr() is what is known as a “string” function and we will examine those in Week 4. For now, just know that substr() takes a character vector as input and then start = and stop = refer to the position within that string that we wish to extract.

Here is a simpler example:

junk <- c("UCLA", "USC", "STANFORD")
substr(junk, start = 1, stop = 2)
## [1] "UC" "US" "ST"

so if you study the code for height, you will see that we are extracting the first number and the last two and performing the appropriate math to convert it into inches.

Now we can finally create our plot:

ggplot(test, aes(x = height, y = weight)) +
    geom_point(data = test, aes(color = Pos)) +
    geom_smooth(method='lm')

Let’s break it down into pieces:

ggplot(test, aes(x = height, y = weight))

This tells ggplot that the data is called test and the aesthetic (aes) informs ggplot about the appearance of the graphic, that is, height on the x-axis and weight on the y-axis. If you only ran that line, there would be no error, but there would be no graphic either.

To generate a graphic in ggplot, one must include a geom_ a geom indicates the display. In this case, we are using geom_point which will yield a scatterplot:

ggplot(test, aes(x = height, y = weight)) +
    geom_point(data = test, aes(color = Pos))

notice the “+” sign. It is used to layer or make additions to the ggplot specifications. See the result of those two lines:

ggplot(test, aes(x = height, y = weight)) +
    geom_point(data = test, aes(color = Pos))

We can add as many geoms as we like to a graphic (but it may start to look terrible…) by using the “+”, so here, we add a regression line.

ggplot(test, aes(x = height, y = weight)) +
    geom_point(data = test, aes(color = Pos)) +
    geom_smooth(method='lm')

The usual question that comes up is “why are there two aes() in this plot” and "why didn’t you put the aes(color = Pos) in the first aes()?

If I had made the choice to put color in the “global aesthetic” (I can’t think of a better description of the first one) we would get this result instead (it’s not wrong, just not what I wished to create):

ggplot(test, aes(x = height, y = weight, color = Pos)) +
    geom_point() +
    geom_smooth(method='lm')

2. Boxplots (one numeric and one non-numeric variable)

Boxplots were developed pre-computers for quick summaries and comparisons. My initial boxplot of 2019-2020 NBA salaries by team is technically correct and informative. However, it is unattractive to the point where a story could be obscured.

ggplot(test, aes(x = Team, y = `Salary2019-20`)) + 
    geom_boxplot()

Suppose you were submitting an application to the NBA for a summer intern-analyst position and one part of the submission requested a demonstration of your visualization skills. Before programming, have a sketch in mind regarding what you could add to the boxplot graphic to make it more informative (e.g., titles) and/or more polished in it’s appearance. Here’s my quick and very dirty… you can use parts of the code for problem 2, but don’t copy it verbatim, the point is to give you a start by showing you some non-intuitive things that can be done with R and ggplot2 (e.g., multiple geoms of the same type, multiple themes):

## we won't be writing our own custom functions until Week 5 but you can see 
## they are handy, I'm writing my own outlier test function
is_outlier <- function(x) {
  return(x < quantile(x, 0.25) - 2 * IQR(x) | x > quantile(x, 0.75) + 2 * IQR(x))
}

## modify the data, recall %>% is interpreted as "then"
## I created separate outliers for the teams DAL and NOP because I needed
## to print them differently to fit them on the graphic
## it is inefficient, but it gives me the "look" I want in little time.
test <- test %>% 
  mutate(final = ifelse(Team %in% c("GSW", "TOR"),"Yes","No"),
         outlier = ifelse(is_outlier(`Salary2019-20`) &
                            !(Team %in% c("DAL", "NOP")), Player, as.numeric(NA)),
         outlier2 = ifelse(is_outlier(`Salary2019-20`) & 
                             Team %in% c("DAL", "NOP"), 
                             Player, as.numeric(NA)))
## now create the boxplot
ggplot(test, aes(x = reorder(Team, `Salary2019-20`, FUN = median), 
                 y = `Salary2019-20`, 
                 fill = final)) + 
    stat_boxplot(geom = "errorbar", width = 0.75, coef = 1.5) + 
    geom_boxplot() +
    scale_fill_manual(values=c("grey", "cyan"), 
                      name = "NBA Final 2019") +
    geom_text(aes(label = outlier), na.rm = TRUE, size = 4, 
              angle = 45, hjust = -0.05, color = "blue") +
    geom_text(aes(label = outlier2), na.rm = TRUE, size = 4, 
              angle = 45, hjust = 0.75, color = "blue") +
    ylim(0, 42000000) +
    labs(title="Salary Distribution by Team, 2019-2020",
        x ="Team", y = "Salary in Dollars") +
    theme_few() +
    theme(plot.title = element_text(hjust = 0.5),
          legend.position="bottom")

AAPL2020 <- AAPL %>%

this data format is also called “stacked data”. For multiple line plots (or multiple anything really) ggplot prefers this structure:

ggplot(data=AAPL2020, aes(x = Date, y = price, color = status)) + 
  geom_line() +
  labs(title="Apple (AAPL) Price in the past 30 days", 
       caption="Source: Yahoo.com finance", 
       y="Price in Dollars") +
  scale_x_date(date_breaks = "days" ,date_labels = "%m/%d") +
  theme_fivethirtyeight() + 
  theme(axis.text=element_text(size=8, angle = 90))
