Combining the two datasets – inner join/merge

The variables used to join each pair of data frames are called keys. A key is a variable (or multiple variables) that can uniquely identify an observation. In our example here, one variable (state) is enough to uniquely identify an observation. Let’s examine the contents of each data set:

library(DT) ## gives us nicely formatted tables for viewing in a web browser
datatable(STmostrecent)

and

datatable(STdensity)

So we have the cases in one dataset and the population in another. Combining these two is easy, we would like the intersection of the two data frames so an inner_join (from dplyr) works well:

library(dplyr)
combined <- inner_join(STmostrecent, STdensity, by = "state")
datatable(combined)

almost there, now we just need to calculate our rate using our combined data:

combined <- combined %>% 
    mutate(infrate = round(cases/(Population/100000), digits = 2)) 
datatable(combined)

The same thing could have been accomplished using base R (and remember, you need to know this method if you are planning to major in Stats). The concepts are the same, the functions are different and the structure of the resulting data is a little different.

Bcombined <- merge(STmostrecent, STdensity, by = "state")
Bcombined$infrate <- round(Bcombined$cases/(Bcombined$Population/100000), digits = 2)
datatable(Bcombined)

We can use our left and right joins to generate more information. We will use some smaller datasets to help us understand more about left and right joins: STjan20.csv, STfeb20.csv and STmar20.csv. These are the states and their cases at the end of January, February, and March 2020.

Left Join

A left join is a situation where we wish to keep all of the observations in the “left hand” (or the first) dataframe in statement. Here is a left join:

STjanEX1 <- left_join(combined[,c(2,11)], STjan20, by = "state")
datatable(STjanEX1)

Notice all of the cases are kept for the “left” data even though there is no match on the “right” side.

Examine a “right join” for the same data:

STjanEX2 <- right_join(combined[,c(2,11)], STjan20, by = "state")
datatable(STjanEX2)

You might think, “wasn’t the result of the right_join the same as performing an inner_join”? In this example yes, but it is not always true. It’s better to think of a right join as keeping all of the “right” side data even if there is no match on the left.
