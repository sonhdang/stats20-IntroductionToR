1. Introduction to Lesson 12

Recall from lecture, a quick summary of factors:

  • Factors are variables in R which take on a limited number of different values (levels), typically these variables are called non-numerical or “categorical” variables.
  • We use factors in statistical modeling because non-numerical variables (e.g., ancestry and it’s “levels” — African, Asian, European, Other) enter into statistical models differently than numerical variables.
  • Designating non-numerical data as factors insures that the modeling functions in R will treat these types of variables correctly.

And why we want to know how to use them:

  • Factors allow changes in the order in which the levels are displayed from their default sorted order, characters have more limitations.
  • A levels= argument can be given a vector for ALL of the possible values of the variable in the order you desire, even if they are not currently present in the data. This is useful during data collection
  • Also, if the ordering of levels is important when performing comparisons (e.g., modeling), use the optional ordered=TRUE argument. In this situation, the factor is known as an ordered factor.

Dates/Times

  • While they have a numeric representation in R (typically, the days or seconds since 1/1/1970), sometimes dates are stored as factors (as opposed to strings) – always check to make sure the function you use converts the factor to character before converting to a datetime (the functions used in this lesson will do that for you).

2. The data

The data was collected by the San Francisco Fire Department and a subset – all calls since 1/1/2020 – was extracted BUT to simplify this lesson, only the unit which arrived on the scene first was kept (in reality, multiple units may be sent to an emergency). There are 72856 rows/observations and 32 fields/variables (please verify this for yourself).

Variables of Interest

Many topics can be examined using the data but from a legal/administrative perspective, the amount of time it takes on average to respond to an emergency may be the most controversial. San Francisco is one of the few cities which makes much of it’s data available publicly. The following may be the most interesting/useful for an analysis:

Received.DtTm - date/time the call was received by 9-1-1 dispatch
Response.DtTm - date/time when a unit responded to the dispatch operator
On.Scene.DtTm - date/time when a unit arrived at the callers location
Hospital.DtTm - if someone needed hospitalization, this is the date/time when a unit arrived at the hospital

Example reading of the data

In lecture, you were shown how to convert a datetime variable into an R DateTime by using a function called as.POSIXct( ) which will convert the datetime string into something both base R and dplyr can use (so you can choose).

For example, here is how you could convert Received.DtTm and save the converted values as a new column on your existing data frame. I’ll call it “START” as for this lesson, this marks the start time of the request for emergency assistance.

Examining Visualizations

When creating visualizations using time variables, something like this could happen – my boxplots are crushed by the outliers:

library(ggplot2)
ggplot(data = FD12, aes(x = Supervisor.District, y = as.numeric(SE1))) +
    geom_boxplot()

or my densities are too highly skewed because of outliers:

ggplot(data = FD12, aes(x = as.numeric(SE1))) +
    geom_density() +
    facet_wrap(~ Supervisor.District)

This “crushing” or “skewness” is too extreme and we lose information. I’ll suggest (a) avoid this information loss if possible and (b) that we might study the common and the extreme separately.

So it turns out that a reponse time difference of more than 600 seconds (about 10 minutes) happens less than 1% of the time, so I’ll exclude those for convenience (this isn’t great data science so if you want to analyze the extremes for this lesson, be my guest). This example shows you how to pass the data directly to ggplot. The period (.) is just a placeholder, it stands for the data which was modified and then passed to ggplot.

FD12 %>%
select(SE1, Supervisor.District, daySTART)  %>%
filter(SE1 <= 600) %>% 
ggplot(data = ., aes(x = Supervisor.District, y = as.numeric(SE1))) +
    geom_boxplot()+
    facet_wrap(~  daySTART >= 77)
